# 歩ける語り場 v033

v033では、v029で発生していた「読み込み中...」のまま止まる問題を修正しました。

## 主な修正

- データ読み込み処理を `loadProjectData()` に整理
- ローカルJSON / GAS API の切替処理を実装
- GAS読み込み失敗時はローカルJSONへフォールバック
- 画像読み込みにタイムアウトと部分失敗許容を追加
- `データ設定` に必要な関数を追加
- データ取得元バッジの更新処理を追加

## 起動

```bash
python -m http.server 8000
```

```text
http://localhost:8000
```


## v033 更新内容

- `editor.html` からGAS WebアプリURLを設定可能
- エディタ側でGAS接続テスト可能
- スプレッドシート由来データをGAS経由でエディタに読み込み可能
- 現在のエディタデータとGASデータの件数差分を確認可能
- GASから読み込んだデータを下書き保存し、ゲームで確認可能

### v033の注意

v033は読み込み専用です。エディタで編集した内容は、まだスプレッドシートへ直接保存されません。


## v033メモ
- ゲーム確認時は、配置エディタの下書きを優先して読み込みます。
- ローカルJSON fetchに失敗しても、同梱fallback-data.jsから起動できます。
- GAS読み込み失敗時もローカル/同梱データにフォールバックします。


## v033追記

GAS読み込み時にスプレッドシート側の maps データが空だった場合、アプリが停止しないように、ローカルJSON/同梱フォールバックデータへ自動復旧する処理を追加しました。Apps Scriptは `gas/Code.gs` をv033版に更新して再デプロイしてください。
