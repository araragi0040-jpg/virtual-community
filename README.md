# 歩ける語り場 v007

v007では、v006の「体験メモ」をもとに、称号・実績・隠し要素のサンプルを追加しました。

## 起動方法

ローカルで直接 `index.html` を開くとJSON読み込みで止まることがあります。簡易サーバーで起動してください。

```bash
cd virtual-community-4u-v007
python -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000
```

## v007で追加したこと

- `data/achievements.json` を追加
  - 実績・称号の条件をデータで編集できます。
- `data/hidden.json` を追加
  - 隠し要素の出現条件・座標・本文をデータで編集できます。
- ヘッダーに「実績」ボタンを追加
- 体験メモに実績数・隠し要素数を表示
- 店内メニューの行動後に条件を満たすと実績が開放
- 条件を満たすと、特定マップに隠し要素が出現

## サンプル実績

- はじめの一歩：何かを1回体験
- 語りの芽：会話ポイント3以上
- 一曲入魂：カラオケ1回以上
- ひと息の達人：休憩ポイント3以上
- 三店舗めぐり：3店舗の店内をすべて訪問
- 小さな発見者：隠し要素を1つ発見
- 街の気配を読む人：隠し要素を3つ発見

## サンプル隠し要素

- 語り場4U店内：会話ポイント3以上で出現
- くさのね劇場店内：カラオケ1回以上で出現
- ブランチカフェ店内：休憩ポイント2以上で出現

## 編集しやすいデータ

### 実績・称号

`data/achievements.json` を編集します。

主な項目：

- `id`：実績ID
- `title`：表示名
- `kind`：`achievement` または `title`
- `description`：開放後の説明
- `lockedHint`：未開放時のヒント
- `condition`：開放条件
- `rewardText`：開放時の表示文

### 隠し要素

`data/hidden.json` を編集します。

主な項目：

- `id`：隠し要素ID
- `mapId`：配置マップ
- `label`：近づいた時の表示名
- `x`, `y`, `w`, `h`, `range`：座標・判定
- `visibleDays`：出現曜日
- `visibleWhen`：出現条件
- `title`：表示タイトル
- `text`：初回発見時の本文
- `foundText`：再確認時の本文
- `log`：体験メモに残るログ
- `stats`：発見時に増える体験値

## 条件タイプ

現在使える条件タイプ：

- `statAtLeast`：特定ステータスが一定以上
- `totalAtLeast`：合計体験値が一定以上
- `actionAtLeast`：特定アクション回数が一定以上
- `visitedMap`：特定マップを訪問済み
- `hiddenFound`：特定隠し要素を発見済み
- `hiddenCountAtLeast`：発見済み隠し要素数が一定以上
- `achievementUnlocked`：特定実績を開放済み
- `dayIs`：特定曜日
- `all`：複数条件すべて
- `any`：複数条件のいずれか
- `not`：条件の否定
