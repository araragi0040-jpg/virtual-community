# v033 GAS読み込み・空データ対策

## 修正概要

v032以前では、GAS APIが応答していても `maps` が0件の場合、アプリ側がその空データを読み込んでしまい、ゲーム画面が「マップデータが空です」で停止することがありました。

v033では以下を修正しています。

- GASレスポンスの `mapsData.maps` が空の場合は、GAS読み込み失敗として扱う
- 失敗時はローカルJSONまたは同梱フォールバックデータへ自動で切り替える
- エディタの下書きプレビューが空データの場合は、下書きを無視して通常データで起動する
- データ設定から「下書きプレビュー解除」ができる
- Apps Script側でも `maps` が0件の場合は `ok:false` として返す

## よくある原因

- スプレッドシートに `maps` シートが存在しない
- `maps` シートにヘッダー行しかない
- v028のTSVを貼り付ける際、シート名が異なっている
- `id` 列が空になっている
- GASを更新した後、Webアプリを再デプロイしていない

## 確認手順

1. スプレッドシートに `maps` シートがあるか確認
2. 1行目にヘッダー、2行目以降にデータがあるか確認
3. Apps Scriptの `gas/Code.gs` をv033版へ更新
4. Webアプリを新しいバージョンで再デプロイ
5. ゲーム画面またはエディタでGAS URLを設定し直して確認
