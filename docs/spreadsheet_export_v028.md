# v028 スプレッドシート移行用CSV/TSV出力

## 目的

v028では、現在JSONで管理している仮想コミュニティアプリのデータを、Googleスプレッドシートに移しやすい形へ展開する。

この段階では、まだGAS連携やスプレッドシートへの自動保存は行わない。まずは、JSONをシート単位のCSV/TSVに分解し、データ構造を確認する段階とする。

## 追加機能

配置エディタに以下を追加した。

- スプシ用TSV一括DL
- CSV一括DL
- シート出力プレビュー

## 推奨形式

Googleスプレッドシートへ貼り付ける場合は、TSVを推奨する。

理由：

- 改行やカンマを含む本文を扱いやすい
- 日本語テキストの貼り付けで崩れにくい
- 1シートごとの列構造を確認しやすい

## 出力シート

| シート名 | 内容 |
|---|---|
| maps | マップ基本情報 |
| interactables | 入口、掲示板、看板、メニュー、イベントなど |
| blocks | 通行不可ブロック |
| npcs | NPC配置・会話ID |
| hidden_spots | 隠し要素 |
| dialogues | 会話本文 |
| confirms | 入店確認などの確認ダイアログ |
| boards | 掲示板本文・リンク |
| menus | 店舗メニュー |
| actions | 店内アクション |
| achievements | 実績・称号 |
| link_boards | リンク集本体 |
| link_items | リンク集内の個別リンク |
| options | 会話・掲示板・メニュー・隠し要素などの選択肢 |

## 運用手順

1. `editor.html` を開く
2. 「データ正規化」を押す
3. 「全体チェック」を押す
4. 問題がなければ「スプシ用TSV一括DL」を押す
5. Googleスプレッドシートに上記シート名でタブを作成する
6. 各TSVを対応するシートに貼り付ける

## 注意点

- `options` シートは、各対象の選択肢をまとめた共通テーブル。
- `parentType` と `parentId` で、どのデータに紐づく選択肢かを判定する。
- JSONオブジェクトや配列は `xxxJson` という列名で保持する。
- v029以降で、スプレッドシートからJSONへ再構成する処理を検討する。

## 次のステップ
