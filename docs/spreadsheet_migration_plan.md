# スプレッドシート移行準備 v015

v015では、現在のJSONデータをGoogleスプレッドシートへ移行しやすいように、`data/sheet_templates/` にCSVテンプレートを出力しています。

## 基本方針

- まずはゲーム内のUI・体験はJSONで検証する
- 固まったデータ構造をCSVテンプレートとして整理する
- その後、Googleスプレッドシートにインポートする
- GAS APIからJSON形式で返し、アプリ側が現在のJSONと同じ形で読み込む

## 座標ルール

全マップは 0〜100 の割合座標で扱います。

- 左上：x=0, y=0
- 右下：x=100, y=100
- w/h：横幅・高さをパーセントで指定

このルールにすることで、画像サイズやスマホ幅が変わっても同じデータを使いやすくします。

## 主なシート候補

| シート名 | 役割 |
|---|---|
| maps | マップ基本情報 |
| interactables | 入口・掲示板・看板・メニューなどの判定エリア |
| transitions | マップ端移動 |
| npcs | NPCの位置・会話ID |
| dialogues | 会話本文 |
| dialogue_options | 会話の選択肢 |
| boards | 掲示板本文とリンク |
| menus | メニュー内容 |
| actions | メニュー行動と結果文 |
| achievements | 称号・実績 |
| hidden_spots | 隠し要素 |
| link_boards | リンク集の見出し |
| link_items | リンク集内のリンク |

## 次の実装候補

1. GoogleスプレッドシートにCSVをインポート
2. GASで `getGameData` を作成
3. アプリ側の `DATA_URLS` をGAS APIへ切り替え
4. 管理画面から配置編集結果をスプシへ保存

## v015の配置エディタについて

`editor.html` は、RPGツクール風ノーコード管理画面の第一歩です。

現時点でできること：

- マップを選ぶ
- 入口・掲示板・看板・メニューの判定位置を動かす
- NPCの位置を動かす
- 隠し要素の位置を動かす
- 変更後の `maps.json` / `npcs.json` / `hidden.json` を保存する

現時点では、保存したJSONを手動で差し替える想定です。将来的にはスプシ/GASに直接保存できるようにします。


## v018追加：blocks

通行不可ブロックは `data/sheet_templates/blocks.csv` で管理する想定です。

| 列 | 内容 |
|---|---|
| id | ブロックID |
| mapId | 配置マップID |
| label | 管理用表示名 |
| x / y / w / h | 0〜100基準の当たり判定範囲 |
| enabled | 表示・有効フラグ |
| visibleDays | 曜日限定ブロックに使う。通常は all |

ゲーム側では、プレイヤー中心点がこの範囲に入る場合は移動不可になります。
